1. Dans PowerShell admin, lance ça
Stop-Service CertSvc -Force -ErrorAction SilentlyContinueImport-Module ADCSDeployment -ErrorAction SilentlyContinueUninstall-AdcsCertificationAuthority -Force -ErrorAction SilentlyContinueRemove-WindowsFeature ADCS-Cert-Authority -IncludeManagementToolsRemove-Item C:\TP-Crypto-ADCS -Recurse -Force -ErrorAction SilentlyContinueRemove-Item "HKLM:\SYSTEM\CurrentControlSet\Services\CertSvc\Configuration" -Recurse -Force -ErrorAction SilentlyContinueRemove-Item "C:\Windows\System32\CertLog" -Recurse -Force -ErrorAction SilentlyContinueRemove-Item "C:\Windows\System32\CertSrv\CertEnroll" -Recurse -Force -ErrorAction SilentlyContinueRestart-Computer
Oui, c’est un peu bourrin, mais sur une VM dédiée au TP, c’est acceptable.
2. Après redémarrage
Relance PowerShell en admin, puis :
cd C:\Users\Administrateur\Desktop\6.3-Gestion-Certifs\windows-adcsSet-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass -Force.\run-on-windows.ps1
Le script réinstallera lui-même le rôle ADCS.
3. Si ça plante encore
À ce stade, ne perds pas 1h : recrée une VM propre ou reviens à un snapshot avant ADCS. Windows ADCS est chiant : une CA partiellement configurée laisse parfois des traces qui bloquent les réinstallations.
Avant de recréer la VM, tu peux quand même m’envoyer le résultat de :
Get-WindowsFeature ADCS-Cert-AuthorityGet-Service CertSvc -ErrorAction SilentlyContinueGet-ChildItem "HKLM:\SYSTEM\CurrentControlSet\Services\CertSvc" -Recurse -ErrorAction SilentlyContinue | Select-Object Name
Mais franchement, si c’est une VM neuve, le plus rapide est souvent : reset VM propre → relancer script une seule fois.
